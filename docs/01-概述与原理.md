# TransPyC 概述与原理

## 1. 项目概述

TransPyC 是一个将 Python 子集代码翻译为 C 代码的源到源编译器，面向操作系统内核等底层开发场景。用 Python 的简洁语法写 C 代码——省去花括号、分号细节，但保留 C 的性能和控制力。

当前版本使用 Rust 实现，核心翻译器约 1500 行，覆盖了函数、结构体、控制流、类型系统、内联汇编等特性。

## 2. 设计目标

1. **简洁性** — Python 语法、C 性能
2. **类型安全** — 通过 Python 类型注解提供完整类型系统
3. **性能等同** — 生成的 C 代码与手写 C 性能一致
4. **底层友好** — 内置内联汇编、指针、宏等底层操作

## 翻译流程

```
Python 源码 (.py)
       ↓
   rustpython-parser 解析
       ↓
   AST (抽象语法树)
       ↓
   第一遍: 收集符号 (collect_symbols)
   - class    → 结构体定义 + 成员类型
   - def      → 函数名
   - x: type  → 变量类型
       ↓
   第二遍: 按序生成 C (generate_c_code)
   - import      → #include
   - class       → struct
   - AnnAssign   → 类型声明
   - FunctionDef → C 函数
       ↓
C 源码 (.c)
```

## 核心组件

### Translator (`core/translator.rs`)

翻译流程的总控制器。持有符号表、作用域栈等状态，提供 `generate_c_code()` 入口方法和各个 AST 节点的处理方法。约 600 行。

### 表达式处理器 (`core/expressions.rs`)

将 Python 表达式 AST 节点转换为 C 代码字符串。处理常量、变量、二元运算、比较、函数调用、属性访问等。也是 `c.*` 和 `t.*` 特殊调用的分发中心。约 500 行。

### 类型名称解析 (`core/type_name.rs`)

递归遍历 Python 类型注解 AST 节点（`t.CInt | t.CPtr` 等），输出 C 类型字符串。处理存储修饰符、指针、数组、结构体等组合规则。约 200 行。

### c 模块 (`includes/gramma.rs`)

`c.Asm()` → `__asm__ volatile`、`c.Memory()` → `((void *)addr)`、`c.Macro()` → `#define` 等底层操作的代码生成。每个 `c.*` 调用对应一个纯函数。

### t 模块 (`includes/types.rs`)

Python 类型名到 C 类型名的映射表。`t.CInt` → `"int"`、`t.CPtr` → `"*"`、`t.CStruct(name="X")` → `"struct X"` 等。

## 4. 类型系统

### 基本类型映射

| Python 类型 | C 类型 | 说明 |
|------------|--------|------|
| `int` | `int` | 整数 |
| `float` | `float` | 32位浮点 |
| `str` | `char*` | 字符串指针 |
| `bool` | `bool` | 布尔 |

### t 模块类型

| Python | C |
|--------|---|
| `t.CChar` | `char` |
| `t.CInt` | `int` |
| `t.CLong` | `long` |
| `t.CFloat` | `float` |
| `t.CDouble` | `double` |
| `t.CVoid` | `void` |
| `t.CPtr` | `*` |
| `t.CStatic` | `static` |
| `t.CExtern` | `extern` |
| `t.CConst` | `const` |

### 类型组合 (`|` 运算符)

```python
# 指针
ptr: t.CInt | t.CPtr               # int*

# 数组
arr: t.CChar[16]                    # char[16]

# 静态变量
counter: t.CStatic | t.CInt       # static int

# 结构体指针
task: t.CPtr | t.CStruct(name="TASK")  # struct TASK*
```

## 4.3  符号表

翻译器维护一个全局符号表，第一遍扫描 AST 时填充：

```text
"MyStruct" → Struct {
    members: {
        "field1": { type: "int", is_pointer: false },
        "ptr":    { type: "struct Other*", is_pointer: true }
    }
}
"my_func"  → Function
"global_x" → Variable { declared_type: "int", is_pointer: false }
```

主要用途：
- 变量声明时判断是新声明还是已存在
- 成员访问时判定用 `.` 还是 `->`
- 从符号表查找变量类型以推断

## 作用域管理 (`VarScopes`)

栈结构，每进入一个函数 push 一层，每退出 pop 一层。查找变量从栈顶向下搜索（内层覆盖外层）。

```text
[ { global vars },              ← 全局作用域
  { param1: "int", ... },        ← 函数参数
  { local_var: "char" } ]        ← 局部变量
```

## `.` vs `->` 运算符选择

翻译器根据变量类型自动选择：

| Python | C | 原因 |
|--------|---|------|
| `s.field` | `s.field` | s 不是指针 |
| `self.field` | `self->field` | self 总是指针 |
| `ps.field` | `ps->field` | ps 声明为指针 |
| `arr[i].field` | `arr[i].field` | 数组元素非指针 |

## 6. 限制与注意事项

1. **不支持 Python 高级特性**：类继承、异常、生成器、装饰器、动态类型
2. **需要显式类型注解**：变量、参数、返回值
3. **内存管理**：需要手动管理内存（malloc/free）
4. **标准库**：不支持 Python 标准库，使用 C 标准库
